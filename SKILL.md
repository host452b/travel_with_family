# Travel Data Skill — CSV→Notebook 转换规范

## CSV→ipynb 转换流程

1. 读取 CSV (utf-8-sig 编码)
2. 按页分割 (50 行/页)
3. 生成 HTML 表格，注入色阶样式
4. 封装为 Jupyter Notebook JSON
5. GitHub 自动渲染

## 色阶规则

### 评分列 (0-10 分)
```
低分(≤5): rgba(220, 60, 60, 0.35)   红色
中等(6-7): rgba(220, 240, 60, 0.35) 黄色
高分(8-10): rgba(40, 200, 100, 0.35) 绿色
```
公式:
```python
ratio = max(0, min(1, (score - 3) / 7))
if ratio < 0.5:
    r, g, b = 220, int(60 + ratio*2*180), 60
else:
    r, g, b = int(220 - (ratio-0.5)*2*180), 200, int(60 + (ratio-0.5)*2*40)
```

### 口碑方差列 (0-5, 反向色阶)
```
稳定(0-1.5): 绿色 → 可信
中等(1.5-2.5): 黄色
分化(2.5-5): 红色 → 两极评价
```

### 不着色的列
- 序号、国家、区域、城市、景区名称、大类、细分类型
- 签证政策、建议时长、日均预算、货币
- 最佳季节、最佳月份、备注、数据来源

## 格式规范

| 元素 | 规则 |
|------|------|
| 字体大小 | 完整数据表 9-10px, 排行榜 11px |
| 景区名称(中) | `<b>` 加粗 |
| 预算列 | 不着色，右对齐 |
| 每页行数 | 50 行 |
| 表格样式 | `white-space: nowrap` |

## 排行榜 Notebook 结构

每个排行榜表格:
1. 标题 `<h3>`
2. 按目标指标降序排列
3. 限制 15-30 行
4. 必须包含列: 景区名称, 国家/区, **性价比**, **综合加权满意度**
5. 根据主题选择额外列 (亲子→儿童年龄, 浪漫→夫妻, 安全→华人安全度)

## 6 个 CSV 统一 34 列

```
序号/区域/国家/省份/城市/景区名称(中)/景区名称(英)
大类/细分类型/中国签证政策
游客口碑/亲子友好度/老人友好度/夫妻友好度/一人游友好度
适合儿童年龄段/儿童出行备注
建议游览时长/日均预算/预算货币
交通可达性/全年适宜度/最佳旅游季节/最佳月份
文化生态独特价值/基础设施与配套/性价比
旅游安全度/华人安全度/口碑方差
综合加权满意度/特色活动与玩法/数据来源
```

## 数据质量要求

- 评分必须在 0-10 范围内
- 每条必须有: 性价比, 口碑方差, 旅游安全度, 华人安全度, 备注
- 新增数据必须多源校准 (携程/TripAdvisor/GPI/Numbeo)
- 安全度按国家级别校准，不按景点
- 口碑方差: 低=评价一致可信, 高=两极分化
- 不同 CSV 中同一景点的评分应一致 (避免重复冲突)
